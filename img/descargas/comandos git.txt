Crear un commit desde un repositorio clonado 

Para crear una rama es 
$ git branch NombredeLaRama

Para moverse a una rama es 
$ git checkout NombredeLaRama

--Siempre trabajar en su rama, hacer el cambio y guardar con crtl+s

--Primero hacer el commit en su rama local

$ git status (aparecerán lineas rojas por los archivos que han modificado o agregado)

$ git add . (para agregar todos los cambios al area de trabajo)

$ git status (para verificar que se hayan agregado correctamente, aparecerán
lineas verdes)

$ git commit -m "Aquí ponen el comentario que veremos todos"

$ git status (mensaje: nothing to commit)

--Luego, fusionar su rama local con la rama main local
--Primero, pararse en la rama main
$ git checkout main

$ git merge NombreDeLaRama

$ git status

--Enviando los cambios al repositorio remoto que se llama origin
$ git push origin main 

--El commit deberia aparecer en GitHub
------------------------------------------------------------------------

Incorporar los últimos cambios si alguien hizo un push

Primero, pararse en la rama main
$ git checkout main

--Tener una vista previa de los nuevos cambios
$ git fetch (para chequear si la rama main local que tenemos está actualizada
con la remota) 

Si no sale nada luego del fetch quiere decir que ya está actualizada
y hacer de frente un $ git pull origin main. Por el
contrario, si salen varias líneas, hacer:

$ git checkout origin/main (te redirige al repositorio remoto para ver una 
vista previa de los cambios pero no los incorpora) --Aqui, solo ver las 
lineas de codigo, no hacer un RunFile, porque luego salen errores jej

$ git checkout main (volviendo a la rama local main)

$ git pull origin main (incorporar los cambios al main local)

--Pararse en su rama para que tenga los cambios
$ git checkout NombreDeLaRama

$ git merge main (fusionar su rama local con el main local)

--Seguir trabajando desde su rama para evitar conflictos:)
